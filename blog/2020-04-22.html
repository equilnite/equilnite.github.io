Design choices for the static SUSA website

The SUSA website, in all honestly is pretty much my first ever full blown out website project. In contrast to this website, it has way more features that are integrated/planned. You might notice that the navigation bar for this website is sort of broken (currently because I'm too lazy to fix it). On the other hand, the navigation bar for the SUSA website is pretty much perfect.

General Design Choices

Fundamentally, I imagined the SUSA website to be different from this personal website of mine. This website is limited to the screen of the device, however the SUSA website is designed so that "blocks" of the page are shown fullscreen and overflow is enabled to scroll through the body of the website. The overflow on this website, is enabled only within containers but this seems to be counter to the "accessibility" mission of SUSA, so I had the entire body scroll instead. And in addition to that accessibility mission, I also added a footer (which is neccessary anyways for OCF hosting recognition) that has a site map and other various links to help redirect users. One big thing that I had to remove from the website to give a static release is a fullcalendar presentation of our public facing SUSA Google Calendar. The API can't be released on GitHub, so I had to unfortunately remove it. One other thing is that I am planning to create a login system for all members, so that materials are secured within SUSA (but obviously it won't prevent all spread of our materials).

You may have also noticed that I avoided the use of Bootstrap or similar libraries. That's because I simply don't want to spend the time to learn them and I am already used to styling by scratch. Anyways, it isn't neccessary to even use those, since there plenty of open-source JavaScript functions on stack-overflow or repositories that can achieve the aesthetic that you want for the website. For example, the landing page of the website used a pure JS version of a jQuery plugin. By also limiting myself to things that browsers themselves can render from just the data that they're sent, I believe that the responsiveness of the website is increased a lot as opposed to if I were to try to learn jQuery/Bootstrap and caused loading issues that make the website unresponsive (some organizations' websites on campus suffer from this).

Making that website with the idea of blocks also made me realize the importance of relative positioning. Since I didn't really need relative positioning on this website, I sort of underestimated using it as opposed to absolute positioning. I then realized that I had to use it in order for my repositioning to work outwell.

Inspirations

I am not that of a creative person, so I drew a lot of the ideas I had for the website from other organizations on campus (I didn't copy code). I thought that a landing element or something similar would look good (took me a long time to find out that fittext existed though), so I added it a long with a link to the id that scrolls to the rest of the content on the page.

After looking through many pages, I personally prefered for pages to also not have too many features, so I wanted something that was a bit more minimalistic but also included some nice features. That's why there isn't any fancy transitions or photo albums (will likely include some slideshows of photos of events/group photos, but nothing that would make the website non-responsive at any time of the loading).

On the team page, there are summary segments for each officer's bio, and that's something that I took from R. R is widely used in our statistics classes, so I decided to mimic a console output to give "summary statistics" of each person (even though the format isn't completely correct).

Conclusion

Overall, I learned a lot about HTML/CSS interactions when making the SUSA website, and applied those learnings to this website. I actually remade all css files for this website and altered the structure of the my html files after I finished the SUSA website. From the team page and the footer, I also learned how to use flex display so that the bios/contents reposition nicely from window resizing.
